## 🚀 快速测试

访问: http://localhost:3000

### 2. 检查API配置

- "VLM提供商"应该自动选择为"MiniMax VL"

### 3. 上传测试图片
- 点击"点击选择文件"
- 选择一张手机截图
- 或者从相册选择多张

### 4. 生成话题
1. 从左侧列表选择一张截图
2. 点击右侧"生成话题"按钮
3. 等待API响应（约5-10秒）
4. 查看生成的话题chips

## 🔍 观察要点

### API响应
- **响应时间**: 5-15秒
- **成功状态**: 应该显示生成的话题
- **失败处理**: 如果失败会使用模拟数据

### 生成质量
- **相关性**: 话题是否与截图相关？
- **准确性**: 是否正确理解了屏幕内容？
- **有用性**: 用户真的会问这个问题吗？
- **自然度**: 表达是否自然？

### 边界情况
- **简单截图**: 白屏、纯文字页面
- **复杂截图**: 多个元素、密集信息
- **特殊内容**: 聊天记录、隐私信息

## ⚠️ 可能的问题

### 问题1：API调用失败
**现象**: 显示"API调用失败，使用模拟数据"
**原因**:
- API Key无效
- 网络问题
- API限流

**解决**:
- 检查API Key是否正确
- 等待一段时间后重试
- 查看浏览器控制台错误信息

### 问题2：响应很慢
**现象**: 等待时间超过15秒
**原因**:
- 图片太大
- API服务器负载高
- 网络延迟

**解决**:
- 使用较小的图片（<1MB）
- 耐心等待
- 检查网络连接

### 问题3：生成质量差
**现象**: 话题不相关或无意义
**原因**:
- Prompt不够优化
- 图片内容太复杂
- 模型理解偏差

**解决**:
- 尝试不同的截图
- 优化Prompt
- 切换到其他VLM提供商
